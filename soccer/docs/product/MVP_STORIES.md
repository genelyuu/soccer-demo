# MVP 유저스토리 (운영 루프 중심)

> 근거: `.claude/tasks/easynext-football-mvp/0002.json`, `.claude/teams/config.json:26`
> 레퍼런스: clubmanager, SCMS, ninety-minute-app, Soccer-Friend

## 핵심 운영 루프

```
일정 생성 → 출석/투표 → 경기 확정 → 기록 입력
(Schedule)   (Attendance)  (Confirm)    (Record)
```

---

## 페르소나

### 운영자 (Manager/Admin)
- 동호회를 생성하고 관리하는 사람
- 경기 일정을 등록하고, 출석을 확인하며, 경기를 확정하고, 기록을 관리한다

### 선수 (Member)
- 동호회에 소속된 일반 회원
- 경기 출석 여부를 투표하고, 확정된 경기에 참가하며, 자신의 기록을 조회한다

---

## 유저스토리

### US-01: 회원가입/로그인
- **운영자/선수**: 이메일/비밀번호로 회원가입하고 로그인할 수 있다
- **수락 기준**: 로그인 후 세션(JWT)이 유지되며, 권한에 따라 다른 UI가 보인다
- 근거: `.claude/tasks/easynext-football-mvp/0004.json`

### US-02: 팀 생성/가입
- **운영자**: 새 팀을 생성할 수 있다 (팀명, 설명, 로고)
- **운영자**: 멤버를 초대하고 역할을 부여할 수 있다 (ADMIN/MANAGER/MEMBER)
- **선수**: 초대를 통해 팀에 가입할 수 있다
- 근거: `.claude/tasks/easynext-football-mvp/0009.json`

### US-03: 경기 일정 등록 (운영 루프 Step 1)
- **운영자**: 경기 일정을 생성할 수 있다 (날짜, 시간, 장소, 상대팀)
- **수락 기준**: 경기 생성 시 출석/투표가 **자동으로** 생성된다
- **운영자/선수**: 경기 목록을 리스트/캘린더 형태로 조회할 수 있다
- 근거: `.claude/tasks/easynext-football-mvp/0005.json`, `0006.json`

### US-04: 출석/투표 (운영 루프 Step 2)
- **선수**: 경기에 대해 참석/불참/보류를 투표할 수 있다
- **운영자**: 투표 현황을 실시간으로 확인할 수 있다
- **수락 기준**: 투표 마감 전까지 변경 가능, 현재 참석 인원 표시
- 근거: `.claude/tasks/easynext-football-mvp/0006.json`

### US-05: 경기 확정 (운영 루프 Step 3)
- **운영자**: 출석 투표를 바탕으로 경기를 확정할 수 있다
- **수락 기준**: 확정 시 **기록실(RecordRoom)이 자동으로 생성**된다
- **수락 기준**: 중복 확정 방지 (멱등성)
- **선수**: 경기 확정 알림을 받고 최종 참석자 명단을 확인할 수 있다
- 근거: `.claude/tasks/easynext-football-mvp/0007.json`

### US-06: 기록 입력 (운영 루프 Step 4)
- **운영자**: 확정된 경기의 기록실에서 스탯을 입력할 수 있다 (득점, 어시스트, 경고/퇴장)
- **운영자**: 경기 메모를 작성할 수 있다
- **선수**: 자신의 기록과 팀 기록을 조회할 수 있다
- 근거: `.claude/tasks/easynext-football-mvp/0008.json`
