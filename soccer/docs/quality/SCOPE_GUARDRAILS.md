## MVP 범위 정의

### 포함 (IN SCOPE)

| 기능 | 유저스토리 | 완료 기준 | 근거 |

| 팀 생성/가입 | US-02 | 팀 CRUD, 멤버 초대, 역할 부여 | `docs/product/MVP_STORIES.md:34-38` |
| 경기 일정 등록 | US-03 | 경기 생성 시 투표 자동 생성 | `docs/product/MVP_STORIES.md:40-44` |
| 출석/투표 | US-04 | 참석/불참/보류, 투표 현황 조회 | `docs/product/MVP_STORIES.md:46-50` |
| 경기 확정 | US-05 | 확정 시 기록실 자동 생성, 멱등성 | `docs/product/MVP_STORIES.md:52-57` |

### 제외 (OUT OF SCOPE) — 반드시 차단

| 기능 | 차단 사유 | 대체 방안 |
|------|---------|----------|
| 알림 시스템 (푸시/이메일) | 외부 서비스 의존, 복잡도 급증 | UI 내 상태 표시로 대체 |
| 캘린더 뷰 | 추가 라이브러리/복잡 UI, MVP 핵심 아님 | 리스트 뷰로 대체 |
| OAuth 소셜 로그인 | 프로바이더별 설정, 심사 필요 | 이메일/비밀번호만 |
| 대결/랭킹 상세 | 도메인 복잡도 급증 | 최소 프로토타입(P2)으로 제한 |
| 팀 간 대결 스키마 | 스키마 변경 범위 과대 | MVP 범위 밖 |
| 실시간 알림 (WebSocket) | 인프라 복잡도, Supabase Realtime 설정 | 폴링 또는 수동 새로고침 |
| 통계 대시보드 | 집계 쿼리/차트 라이브러리 | 기록실 조회로 대체 |
| 파일 업로드 (프로필/팀 로고) | 스토리지 설정, 리사이징 | URL 입력만 허용 |
| 다중 팀 전환 UI | UX 흐름 미정의 | 단일 팀 가정 |
| 국제화 (i18n) | MVP에 불필요한 복잡도 | 한국어 단일 |

## 스코프 팽창 경고 기준

다음 조건에 해당하면 **스코프 팽창**으로 판단하고 차단한다:

1. **OUT OF SCOPE 목록에 포함된 기능** 구현 시도
2. 기존 테이블에 **3개 이상 컬럼 추가** 또는 **새 테이블 생성** 시 ADR 없이 진행
3. **새 외부 의존성**(npm 패키지) 추가 시 근거 문서 없이 설치
4. 단일 PR에서 **5개 이상 파일 신규 생성**
5. 테스트 없이 **도메인 로직 변경**
6. [UNKNOWN] 항목을 근거 없이 임의로 결정

## 수정 통과 조건

PR이 병합되려면 다음을 충족해야 한다:

1. MVP IN SCOPE 범위 내 변경인가?
2. 관련 ADR 또는 문서 근거가 있는가?
3. 테스트가 추가/수정되었는가? (도메인 로직 변경 시 필수)
4. OUT OF SCOPE 기능이 포함되어 있지 않은가?
5. `npm run typecheck && npm run lint && npm run test` 통과하는가?

## 검증 체크리스트

- [ ] 현재 진행 중인 작업이 IN SCOPE 목록에 포함되는가?
- [ ] OUT OF SCOPE 기능이 슬금슬금 추가되고 있지 않은가?
- [ ] 새 패키지 설치 시 ADR에 근거가 기록되었는가?
- [ ] [UNKNOWN] 항목이 무단으로 확정 처리되지 않았는가?
