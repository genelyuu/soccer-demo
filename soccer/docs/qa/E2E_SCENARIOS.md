# E2E 시나리오 (Task 0020)

> 근거: `.claude/tasks/easynext-football-mvp/0020.json`

## 시나리오 1: 운영자 — 경기 생성부터 기록 완료까지

```
1. 로그인 (/auth/signin, manager@demo.com)
2. 경기 목록 (/matches) 진입
3. [새 경기] 클릭 → /matches/new
4. 경기명: "주말 리그전", 일시: 다음 주 토요일 14:00, 장소: "잠실 운동장"
5. [경기 등록] 클릭
6. → /matches로 리다이렉트, 새 경기 카드 확인
7. 경기 카드 클릭 → /matches/[id]
8. 출석 투표 현황 확인 (전원 PENDING)
9. [경기 확정] 클릭 → 상태가 CONFIRMED로 변경
10. [기록실 바로가기] 클릭 → /matches/[id]/record
11. 선수별 스탯 입력 (득점 2, 어시스트 1)
12. [저장] 클릭 → 기록 테이블에 반영
```

**예상 결과**: 경기 생성 → 투표 자동 생성 → 확정 → 기록실 자동 생성 → 기록 입력 완료
**검증 포인트**: 운영 루프 4단계가 끊김 없이 진행

## 시나리오 2: 선수 — 투표 후 기록 조회

```
1. 로그인 (/auth/signin, player1@demo.com)
2. 경기 목록 (/matches) 진입
3. OPEN 상태 경기 클릭 → /matches/[id]
4. [참석] 버튼 클릭
5. 투표 상태가 ACCEPTED로 변경 확인
6. (운영자가 경기 확정 후)
7. 경기 상세에서 상태가 CONFIRMED 확인
8. [기록실 바로가기] → /matches/[id]/record
9. 자신의 기록 조회
```

**예상 결과**: 투표 성공, 확정 후 기록 조회 가능
**검증 포인트**: 선수 권한으로 기록 입력 불가, 조회만 가능

## 시나리오 3: 회원가입 후 팀 가입

```
1. /auth/signup 진입
2. 이름: "새 선수", 이메일: "new@test.com", 비밀번호: "password123"
3. [회원가입] 클릭 → 자동 로그인 → / 리다이렉트
4. /team 진입 → "소속된 팀이 없습니다"
5. (운영자가 이메일로 초대)
6. /team 새로고침 → 팀 카드 표시
```

**예상 결과**: 회원가입 성공, 초대 후 팀 소속 확인
**검증 포인트**: 가입→로그인 자동 전환, 팀 초대 플로우

## 시나리오 4: 권한 없는 접근 시도

```
1. MEMBER 역할로 로그인
2. /matches/new 직접 접근 → 폼 표시됨
3. 경기 생성 시도 → 403 에러 ("경기 생성 권한이 없습니다")
4. OPEN 경기에서 [경기 확정] 시도 → 403 에러
```

**예상 결과**: UI는 접근 가능하나 API에서 권한 차단
**검증 포인트**: 403 에러가 적절히 표시되는가

## 시나리오 5: 에지 케이스 — 중복 확정/마감 후 투표

```
1. 운영자로 경기 확정
2. 다시 [경기 확정] 시도 → 멱등하게 처리 (기존 기록실 반환)
3. 선수로 로그인
4. CONFIRMED 상태 경기에서 투표 시도 → "투표가 마감되었습니다"
5. 기록실 CLOSED 후 기록 입력 시도 → "기록실이 마감되었습니다"
```

**예상 결과**: 중복 작업이 안전하게 처리됨
**검증 포인트**: 멱등성, 상태 제약

## 실패 케이스 재현 절차

| 실패 유형 | 재현 방법 | 예상 에러 |
|----------|---------|---------|
| 인증 없이 API 호출 | 로그아웃 후 /api/matches GET | 401 |
| 잘못된 이메일 회원가입 | "invalid-email" 입력 | 400 (유효한 이메일을 입력해주세요) |
| 짧은 비밀번호 | "123" 입력 | 400 (최소 6자) |
| 권한 없는 경기 생성 | MEMBER로 POST /api/matches | 403 |
| 취소된 경기 수정 | CANCELLED 경기 PATCH | 409 |
| 존재하지 않는 경기 | /matches/invalid-uuid | 404 |
